## 第二階段：安全性與架構修正

本次修正 6 項嚴重安全與架構問題。

### 9. 移除 `/maps/import` 端點

- [x] 刪除 `app/main.py` 中的 `/maps/import` POST 端點
- 該端點允許任意讀取伺服器檔案（路徑遍歷風險），且啟動時已自動載入地圖，無實際用途

### 10. 修正 WebSocket 硬編碼 URL

- [x] `frontend/src/hooks/useWebSocket.js` 改為動態推算 WebSocket URL
- 修正前：`ws://localhost:8000/ws`（僅能在本機開發使用）
- 修正後：`${protocol}//${window.location.host}/ws`（自動適應任何環境）
- 移除多餘的 `console.log` 除錯訊息

### 11. `/orders` POST 加入認證

| 檔案 | 修改內容 |
|------|---------|
| `app/main.py` | `create_order` 新增 `Depends(get_current_user)`，使用 `current_user.email` 取代前端傳入值 |
| `frontend/src/api/orders.js` | `createOrder` 新增 `token` 參數，帶入 `Authorization` header |
| `frontend/src/pages/Checkout.jsx` | 從 authStore 取 token 傳入，移除 `orderInfo.userEmail` |

### 12. `/orders/{order_id}` GET 加入認證

| 檔案 | 修改內容 |
|------|---------|
| `app/main.py` | `get_order` 新增 `Depends(get_current_user)`，檢查 `order.user_email == current_user.email`，不符回 403 |
| `frontend/src/api/orders.js` | `getOrder` 新增 `token` 參數 |
| `frontend/src/pages/Tracking.jsx` | 從 authStore 取 token 傳入 |

### 13. 收緊 CORS 設定

- [x] `app/main.py`：改用 `ALLOWED_ORIGINS` 環境變數（預設 `http://localhost:5173,http://localhost:8000`）
- [x] `docker-compose.yml`：backend environment 新增 `ALLOWED_ORIGINS=http://localhost:5173,http://robot-frontend:5173`
- 修正前：`allow_origins=["*"]`（允許任何來源）
- 修正後：僅允許明確指定的來源

### 14. 統一前端 API Base 路徑

- [x] `frontend/src/stores/authStore.js`：`API_BASE` 從 `''` 改為 `'/api'`
- 所有前端 API 呼叫統一走 `/api` 前綴，透過 vite proxy rewrite 規則轉發至後端
